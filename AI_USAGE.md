# AI 自动报告功能使用指南

## 1. AI 自动报告功能是什么

AI 自动报告功能让你在完成数据上传和统计分析后，直接调用大语言模型（LLM）自动撰写数据分析报告。

**工作流程：**

```
上传数据 → 变量识别 → 分析配置 → 统计计算 → 打包 Payload → 发送 LLM → 生成报告
```

关键点：**AI 只负责解释统计结果和撰写报告文字，所有统计计算由程序完成。**

---

## 2. 为什么不直接把原始数据发给 AI

1. **隐私安全**：原始数据可能包含姓名、手机号、身份证号等个人身份信息，直接发送给 AI 服务商存在隐私风险。
2. **数据量过大**：LLM 的上下文窗口有限，原始数据动辄几千行，远超处理能力。
3. **统计准确性**：Python 的统计计算比 LLM 自己算要准确和可靠得多。
4. **结果可复现**：程序计算的统计结果可复现，LLM 直接计算可能每次结果不同。

因此，本平台只发送 **统计摘要**（均值、频数、相关系数、回归系数等），不发送原始数据行。

---

## 3. analysis_payload 是什么

`analysis_payload` 是程序将统计分析结果打包成的结构化 JSON 数据，发送给 LLM 用于报告撰写。

**设计原则：「固定结构 + 动态内容」**

- 外层结构固定（8 个 section），LLM 可以稳定解析
- 内部变量名、分析类型、统计结果 100% 来自用户上传的数据
- 不写死任何特定变量名（如 `overall_satisfaction`）或场景措辞（如「政务服务中心」）

**Payload 结构：**

```json
{
  "project_meta": {},           // 报告标题、研究对象、生成时间
  "data_overview": {},          // 样本量、变量数、缺失值
  "variable_schema": [],        // 每个变量的类型、隐私设置
  "user_analysis_config": {},   // 目标变量、分组变量、排除变量
  "analysis_plan": [],          // 所有计划执行的分析
  "analysis_results": [],       // 统一分析结果列表
  "chart_summaries": [],        // 图表文字摘要（不含图片）
  "warnings": []                // 警告和注意事项
}
```

### Payload 不包含什么

- ❌ 完整原始数据行
- ❌ API Key
- ❌ 未授权发送的高风险隐私明细
- ❌ 图片

---

## 4. 如何生成 analysis_payload

1. 在「通用模式」下上传数据。
2. 完成变量识别和分析配置（设置目标变量、分组变量、解释变量）。
3. 进入「AI 自动报告」标签页。
4. 在「隐私与变量使用设置」中确认隐私配置。
5. 点击「生成 Payload」按钮。
6. 等待程序自动运行所有统计分析并打包为 JSON。

Payload 生成后可以预览摘要信息、查看完整 JSON、或下载 `analysis_payload.json`。

---

## 5. 如何配置 API Key

### 方法一：页面直接输入
在「AI 自动报告」标签页的「API Key」输入框中直接输入 Key（输入框为密码模式，不会显示明文）。

### 方法二：环境变量
根据厂商的 `api_key_env` 设置对应的环境变量，例如：
- OpenAI: `OPENAI_API_KEY`
- 小米 MiMo: `MIMO_API_KEY`

### 方法三：Streamlit Secrets
在 `.streamlit/secrets.toml` 中配置：
```toml
OPENAI_API_KEY = "sk-your-key-here"
MIMO_API_KEY = "your-mimo-key-here"
```

## 6. 如何使用内置厂商

平台内置了多个 LLM 厂商配置，在 `config/llm_providers.yaml` 中定义：

1. 选择「AI 厂商」下拉框中的厂商。
2. 输入 API Key。
3. 选择或手动输入模型名。
4. 点击「测试连接」验证配置是否正确。

支持的协议类型：
- `openai_compatible` — 标准 OpenAI Chat Completions API
- `custom_openai_compatible` — 用户自定义 OpenAI 兼容 API
- `gemini_native` — Google Gemini 原生 API

---

## 7. 如何使用自定义 OpenAI Compatible API

选择厂商为「自定义 OpenAI Compatible API」，可以配置：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 自定义厂商名称 | 显示名称 | 自定义 API |
| Base URL | API 基础地址 | 从配置文件读取 |
| Chat Path | Chat Completions 路径 | `/chat/completions` |
| Auth Type | Bearer / api-key header | bearer |
| Header Name | 鉴权头名称 | Authorization |
| Prefix | 鉴权前缀 | Bearer |
| 模型列表路径 | 获取模型列表的路径 | `/models` |
| 响应格式 | openai_models / generic_models | openai_models |

常见第三方 API 配置示例：

**DeepSeek：**
- Base URL: `https://api.deepseek.com/v1`
- Auth: Bearer, Header: Authorization, Prefix: Bearer

**Ollama（本地）：**
- Base URL: `http://localhost:11434/v1`
- Auth: Bearer（Ollama 默认不需要，可留空 API Key）

## 9. 如何刷新模型列表

对于支持模型列表接口的厂商（在配置中 `model_list.enabled: true`）：

1. 输入 API Key 后，「联网获取模型列表」按钮变为可用。
2. 点击按钮，系统向厂商 API 请求最新模型列表。
3. 成功后，模型下拉框自动填充，选择即可。
4. 模型列表会缓存到 `cache/model_catalog.json`（缓存 1 小时）。

---

## 10. 刷新模型列表失败怎么办

1. **使用缓存**：如果之前成功获取过模型列表，缓存会自动加载（页面会显示更新时间）。
2. **手动输入**：即使获取失败，你仍然可以手动输入模型名。在下拉框下方的输入框中直接输入模型名即可。
3. **不影响功能**：模型列表获取失败不影响报告生成，只要模型名正确。

常见失败原因：
- API Key 错误 → 检查 Key 是否正确
- 网络问题 → 检查能否访问厂商 API
- 厂商不支持模型列表接口 → 直接手动输入模型名

## 11. 如何选择报告结构类型

在「报告参数设置」中，选择「报告结构类型」：

| 结构类型 | 章节数 | 适用场景 |
|----------|--------|----------|
| **通用调研报告** | 8 章 | 一般性问卷数据分析、初学者友好 |
| **学术论文式报告** | 摘要+关键词+7 章 | 课程论文、学术研究、正式报告 |
| **政务决策报告** | 6 章 | 公共服务、政策分析、政府调研 |
| **商业分析报告** | 7 章 | 商业数据分析、业务决策支持 |
| **课程作业报告** | 7 章 | 课堂展示、教学评估 |

**学术论文式报告**包含摘要、关键词、研究背景、研究方法、实证结果、讨论、结论与局限等完整学术论文结构。

## 14. Markdown、HTML、Word 报告分别适合什么场景

| 格式 | 特点 | 适用场景 |
|------|------|----------|
| **Markdown** | 纯文本、易编辑、可版本管理 | 进一步编辑、GitHub/GitLab、技术文档 |
| **HTML** | 带样式、可浏览器查看、支持主题 | 网页展示、邮件附件、快速预览 |

## 15. 隐私与变量使用设置说明

平台采用「**系统识别风险 → 用户确认使用**」的隐私模型，而不是「一旦疑似个人信息就强制排除」。

### 隐私风险等级

| 等级 | 说明 | 示例变量 |
|------|------|----------|
| **none** | 无隐私风险 | 通用数值变量 |
| **low** | 低风险（人口统计属性） | age, gender, education, region, income_level |
| **medium** | 中等风险 | 自由文本、详细地址、敏感属性 |
| **high** | 高风险 | 姓名、手机号、身份证号、邮箱 |

### 隐私分类

| 分类 | 说明 |
|------|------|
| `demographic_attribute` | 人口统计属性（年龄、性别、学历等） |
| `contact_info` | 联系方式（手机、邮箱等） |
| `direct_identifier` | 直接身份标识（姓名、身份证号、学号等） |
| `location_info` | 地理位置（详细地址等） |
| `free_text` | 自由文本（开放题答案等） |
| `sensitive_attribute` | 敏感属性（收入具体数值、健康信息等） |
| `financial` | 金融信息（银行卡号等） |

## 16. 高风险变量如何处理

系统对高风险变量（如姓名、手机号、身份证号）的默认处理：

1. **可以用于本地统计**：缺失值检查、唯一值统计等仍然可用。
2. **默认不作为分组变量**：不参与交叉分析。
3. **默认不进入模型**：不参与相关分析和回归分析。
4. **默认不发送给 AI**：原始明细不进入 analysis_payload。

### 四种发送方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **不发送（exclude）** | payload 中完全排除该变量的信息 | 高风险个人身份信息 |
| **仅发送聚合统计（aggregate_only）** | 仅发送频数、均值等聚合结果 | 人口统计变量（默认） |
| **发送脱敏样例（masked_examples）** | 样例值经过脱敏处理 | 需要展示数据特征但保护隐私 |
| **完整发送（full）** | 发送所有可用信息 | 已脱敏的数据或低风险变量 |

### 脱敏示例

| 类型 | 原始值 | 脱敏后 |
|------|--------|--------|
| 手机号 | 13812345678 | 138\*\*\*\*5678 |
| 邮箱 | zhangsan@example.com | z\*\*\*n@example.com |
| 身份证号 | 110101199001011234 | 110101\*\*\*\*1234 |
| 中文姓名 | 张三 | 张\*\* |
| 英文姓名 | John | J\*\*\* |
| 银行卡 | 6222021234567890 | \*\*\*\*7890 |

选择「完整发送」时，系统会弹出二次确认提示，要求你确认数据已脱敏或有权发送。

## 17. 常见错误排查

### 没有 API Key

- **现象**：页面提示「未检测到 API Key」
- **解决**：在输入框中填写 API Key，或设置对应的环境变量

### API Key 错误

- **现象**：测试连接失败，错误信息包含「401」或「unauthorized」
- **解决**：检查 Key 是否正确、是否过期、是否有余额

### 模型名错误

- **现象**：测试连接或生成报告时报错「model not found」
- **解决**：检查模型名拼写（区分大小写），或点击「联网获取模型列表」确认可用模型

### Base URL 错误

- **现象**：连接失败，错误信息包含「Connection refused」或「Name resolution」
- **解决**：检查 Base URL 是否正确（注意 `https://` 和末尾不要有多余的 `/v1`）

### 模型列表刷新失败

- **现象**：点击「联网获取模型列表」后提示错误
- **解决**：
  1. 确认 API Key 正确
  2. 确认厂商支持模型列表接口
  3. 手动输入模型名（不影响报告生成）

### analysis_payload 太大

- **现象**：页面提示「Payload 较大，AI 处理时间可能较长」或「已自动截断」
- **解决**：
  1. 减少变量数量或分析配置
  2. 选择更简短的分析范围
  3. 对于截断情况，关键统计结果通常仍会保留

### Word 报告无法生成

- **现象**：点击下载 DOCX 后无反应
- **解决**：安装 `python-docx`：`pip install python-docx`

### HTML 样式没有变化

- **现象**：切换 HTML 主题后样式相同
- **原因**：HTML 导出是生成报告时确定主题的，之后修改主题需要重新生成报告

### 报告中出现旧项目措辞

- **现象**：AI 报告中出现了「政务服务中心」「公众满意度」等固定措辞
- **原因**：通常是因为某些数据集的变量名恰好包含这些词。如果确认不是，请检查 `analysis_payload` 中 project_meta 和 variable_schema 的内容
- **解决**：修改报告标题和变量中文名为适合当前数据集的名称

### AI 编造了不存在的结论

- **现象**：报告中出现了 data 中没有的数据或显著性结果
- **原因**：LLM 有时会产生幻觉
- **解决**：
  1. 降低 Temperature（建议 0.1-0.3）
  2. 检查 payload 是否完整
  3. 重新生成报告
  4. 人工审阅并修正

## 18. 安全说明

- API Key **不会**写入日志文件
- API Key **不会**写入 `cache/model_catalog.json`
- API Key **不会**写入 `analysis_payload`
- API Key **不会**写入任何格式的报告（Markdown/HTML/Word）
- 错误信息中如果包含 API Key，会自动脱敏
- HTML 报告不包含外部脚本
- HTML 报告可离线查看

## 19. 技术架构

```
app.py (Streamlit UI)
  ├── src/data_loader.py      — 数据加载、工作表选择
  ├── src/schema_infer.py     — 变量类型推断 + 隐私风险评估
  ├── src/generic_analysis.py  — 统计计算（描述统计、交叉分析、相关、回归）
  ├── src/generic_charts.py   — 图表生成（Plotly）
  ├── src/analysis_packager.py — 分析结果打包为 JSON Payload
  ├── src/llm_prompts.py      — AI 提示词模板（系统提示词 + 用户提示词）
  ├── src/llm_client.py       — LLM API 调用（支持多种协议）
  ├── src/ai_report_generator.py — AI 报告生成编排
  ├── src/html_report_templates.py — HTML 报告模板（5 套 CSS 主题）
  ├── src/model_registry.py   — 模型列表获取与缓存
  └── src/provider_config.py  — LLM 厂商配置加载
```

## 20. 快速开始

1. **安装依赖**：`pip install -r requirements.txt`
2. **启动**：`streamlit run app.py`
3. **上传数据**：支持 Excel (.xlsx/.xls) 和 CSV
4. **选择工作表**（Excel 文件）
5. **查看变量识别结果**：确认或调整变量类型
6. **配置分析参数**：设置目标变量、分组变量、解释变量
7. **选择报告参数**：结构类型、写作风格、篇幅、HTML 主题
8. **配置 AI**：选择厂商、输入 API Key、选择模型
9. **生成 Payload**：点击「生成 Payload」打包统计结果
10. **生成报告**：点击「生成 AI 分析报告」
11. **下载**：Markdown / HTML / Word
