# AI 辅助功能使用指南

本文档说明 CivicSurvey Studio（问策 Insight）中的 AI 辅助功能及其使用方法。

---

## 1. AI 功能是可以选的

CivicSurvey Studio 的核心统计分析功能完全离线运行，不需要任何 API Key。

AI 辅助功能是可选的增强：

- **AI 推荐分析方案** — 根据变量结构推荐目标变量和解释变量（需 API Key）
- **AI 报告生成** — 根据统计分析结果自动撰写报告草稿（需 API Key）
- **文献综述检索** — 检索相关学术文献注入报告背景（需 API Key）

**本地统计分析**（描述统计、交叉分析、回归、可视化、模板报告导出）**不需要 API Key**，完全在本地完成。

---

## 2. AI 的工作方式

```
上传数据 → 变量识别 → 分析配置 → 统计计算 → 打包 Payload → 发送 LLM → 生成报告草稿
```

关键点：**AI 只负责解释统计结果和撰写报告草稿，所有统计计算由程序完成。**

AI 推荐方案只是建议，不自动替代人工判断。AI 报告是草稿，需人工复核。

---

## 3. 为什么不直接把原始数据发给 AI

1. **隐私安全**：原始数据可能包含个人身份信息，直接发送存在隐私风险。
2. **数据量过大**：LLM 的上下文窗口有限，原始数据动辄几千行，远超处理能力。
3. **统计准确性**：Python 的统计计算比 LLM 自己算要准确和可靠得多。
4. **结果可复现**：程序计算的统计结果可复现，LLM 直接计算可能每次结果不同。

因此，本平台只发送 **统计摘要**（均值、频数、相关系数、回归系数等），不发送原始数据行。

---

## 4. 发送给 AI 的内容经过隐私过滤

在数据发送给 AI 之前，系统通过 `filter_payload_for_ai()` 进行隐私过滤：

- 被标记为「不发送 AI」的变量会从 payload 中完全移除
- 过滤覆盖 9 个 payload 节（变量列表、分析结果、warnings、metadata 等）
- `aggregate_only` 模式只发送聚合统计，不含原始样例值
- `send_to_ai_mode="none"` 的变量不会进入 AI prompt

详见 [安全说明](docs/security.md)。

---

## 5. 如何配置 API

在侧边栏「AI 设置」expander 中：

1. 选择 LLM 厂商（OpenAI / DeepSeek / Moonshot / SiliconFlow / 自定义 API）
2. 输入 API Key
3. 选择或手动输入模型名
4. 可选：点击「测试连接」验证配置

也可在「报告工作台」→「AI 报告」页面配置。

---

## 6. 配置保存

勾选「记住设置」后，API Key 将以明文保存在 `config/user_settings.json` 中。

- 该文件已被 `.gitignore` 排除，不会被 Git 跟踪
- 个人设备上可以使用此功能方便使用
- 不要在公共或共享设备上使用「记住设置」
- 真实 API Key 不应提交到仓库

---

## 7. 支持的 LLM 厂商

平台通过 `config/llm_providers.yaml` 配置了多家厂商：

- **DeepSeek** — OpenAI 兼容 API
- **OpenAI** — OpenAI Chat Completions API
- **Moonshot（月之暗面）** — OpenAI 兼容 API
- **SiliconFlow（硅基流动）** — OpenAI 兼容 API
- **Google Gemini** — Gemini 原生 API
- **自定义 OpenAI Compatible API** — 任意 OpenAI 兼容接口

## 10. 常见问题

### 没有 API Key

不影响统计分析。描述统计、交叉分析、回归、图表、模板报告全部不需要 API Key。

### API Key 错误

测试连接返回 401 或 unauthorized → 检查 Key 是否正确、是否过期、是否有余额。

### 模型名错误

测试连接返回 model not found → 检查模型名拼写，或点击「联网获取模型列表」确认可用模型。

### AI 编造了不存在的结论

LLM 有时会产生幻觉。建议：
1. 降低 Temperature（建议 0.1-0.3）
2. 检查 payload 是否完整
3. 重新生成报告
4. 人工审阅并修正

### 文献检索为空

文献检索（Semantic Scholar / OpenAlex / CrossRef）以英文数据库为主，中文文献支持有限。不勾选文献综述功能不会影响报告生成。

## 11. 安全说明

- API Key **不会**写入日志文件
- API Key **不会**写入 `cache/model_catalog.json`
- API Key **不会**写入 `analysis_payload`
- API Key **不会**写入任何格式的报告（Markdown/HTML/DOCX）
- 错误信息中如果包含 API Key，会自动脱敏
- HTML 报告可离线查看

详见 [安全说明](docs/security.md)。运行以下命令检查仓库中是否有泄露的 secrets：

```bash
pytest tests/test_no_secrets_committed.py -q
```
