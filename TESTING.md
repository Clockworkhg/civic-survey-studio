# 测试指南

本文档指导你使用 `tests/test_data/` 中的模拟数据，按四类场景依次验证 CivicSurvey Studio（问策 Insight）的全部核心功能。

## 前置准备

### 1. 启动应用

```bash
pip install -r requirements.txt
streamlit run app.py
```

浏览器访问 `http://localhost:8501`。

### 2. 切换模式

在左侧边栏顶部将模式从「示例模式（政务满意度）」切换为 **「通用模式（任意问卷数据）」**。

### 3. 测试数据位置

所有测试数据位于项目根目录下的：

```
tests/test_data/
├── 01_课程教学评价数据_含变量说明.xlsx      (17.6 KB)
├── 02_博物馆观众体验调查_无变量说明.xlsx    (19.5 KB)
├── 03_员工满意度调查_含缺失和英文变量.xlsx   (16.3 KB)
├── 04_电商用户转化行为_无变量说明.csv       (16.1 KB)
└── 测试说明.md
```

> 建议：你也可以将上面的文件复制到新建的 `sample_data/` 文件夹中统一管理。

## 验收标准

平台必须满足以下最低标准方可视为通过测试：

### 数据兼容性
- [x] **Excel 可读取**：`.xlsx` 和 `.xls` 格式均可正常加载
- [x] **CSV 可读取**：自动检测编码（utf-8、gbk、gb2312 等），中文内容不出现乱码
- [x] **不固定工作表名**：系统自动列出所有工作表供用户选择，不依赖特定名称（如 `SPSS_Data`）
- [x] **不固定变量名**：不要求数据包含特定命名的变量（如 `overall_satisfaction`），完全由用户配置

### 变量类型自动推断
- [x] **ID 识别**：唯一值占比 ≥ 98% 或含 ID 关键词的变量被识别为 id 并排除
- [x] **日期识别**：日期关键词 + 可解析验证，或 pandas datetime64 类型
- [x] **数值识别**：连续型数值变量正确分类为 numeric
- [x] **有序识别**：量表模式（1-5、1-7、1-10 等）正确分类为 ordinal
- [x] **分类识别**：唯一值少的变量（≤12 个数值型、≤20 个文本型）正确分类为 categorical
- [x] **二分类识别**：只有 2 个取值的数值变量（如 0/1）识别为 categorical 而非 ordinal
- [x] **文本识别**：含开放题关键词（comment、建议、反馈等）或唯一值极多的变量识别为 text 并排除
- [x] **乱码容错**：变量说明表列名乱码时，系统能通过位置回退正确解析

### 统计方法匹配
- [x] **数值变量** → 描述统计（均值、标准差、中位数、分位数、偏度、峰度）
- [x] **分类变量** → 频数分析、柱状图、卡方检验
- [x] **有序变量** → 频数分析 + 均值/中位数
- [x] **cat × num** → 分组均值比较 + ANOVA/t-test
- [x] **cat × cat** → 交叉表 + 卡方检验
- [x] **num × num** → Pearson r + Spearman ρ + 散点图
- [x] **连续目标** → OLS 多元线性回归
- [x] **二分类目标** → 跳过 OLS，提示逻辑回归，回退到卡方检验/交叉分析

### 可视化
- [x] 系统能根据变量类型自动选择图表类型（直方图/柱状图/箱线图/散点图/热力图）
- [x] 仪表盘自动生成并覆盖目标变量、分组变量、解释变量的相关图表
- [x] 图表标题使用变量说明表中的中文含义，无变量表时使用字段名

### 报告生成
- [x] HTML 报告可正常生成并包含完整章节
- [x] DOCX 报告可正常生成并可用 Microsoft Word 打开
- [x] 报告内容完全数据驱动，**不含任何硬编码的特定调查内容**（如"政务服务中心""公众满意度""district"等）
- [x] 报告中包含"相关不等于因果"等统计学谨慎性声明

## 常见问题排查

### Q1：上传后没有数据 / 预览为空

**可能原因与解决方法：**

1. **工作表名不匹配**：通用模式下系统默认选择第一个工作表。如果数据不在第一个工作表，在侧边栏「工作表」下拉框中手动选择正确的工作表。
2. **表头行设置错误**：如果数据表头不在第一行（例如前几行是注释），调整「表头行」参数（0=第一行，1=第二行，以此类推）。
3. **文件编码问题（CSV）**：如果上传 CSV 后出现乱码，系统会自动尝试 utf-8 → gbk → gb2312 → gb18030 编码链。如果仍乱码，请用记事本将 CSV 另存为 UTF-8 编码后重新上传。
4. **文件格式不支持**：确认文件扩展名为 `.xlsx`、`.xls` 或 `.csv`。

### Q2：变量类型识别错误

**常见误判与手动修正：**

| 误判现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 年龄(age)被识别为 ordinal | 数值范围巧合匹配量表模式 | 在变量说明表中显式指定类型为 `numeric` |
| 0/1 二分类被识别为 ordinal | 两值恰好连续（0,1）且匹配 (0,5) 量表范围 | 在变量说明表中显式指定类型为 `categorical` |
| 评论字段被识别为 categorical | 唯一值少时可能走"少量分类"路径 | 确保字段名含 `comment`/`建议`/`反馈` 等关键词，或在变量说明表中指定 `text` |
| ID 未识别 | 字段名不含 ID 关键词且不满足唯一比 | 在变量说明表中指定类型为 `id` |

> **最佳实践**：如果数据包含变量说明表，其中指定的类型将覆盖自动推断。建议对有歧义的变量手动指定类型。

### Q3：回归分析报错

**常见报错与解决方法：**

1. **`ValueError: NaN in data`**：解释变量含缺失值。系统会自动使用 `dropna()` 排除含缺失值的行，样本量会减小（Tab 7 会显示实际使用的 n）。
2. **`目标变量为分类变量，不适合线性回归`**：如果核心结果变量是分类/二分类变量（如 0/1 是否购买），系统会自动跳过 OLS 并提示使用逻辑回归。这是**预期行为，不是 bug**。
3. **`解释变量不足（至少需要 2 个数值型变量）`**：检查分析配置中是否选择了足够多的数值型或有序型解释变量。
4. **变量不在数据中**：检查配置中选择的变量名是否与数据中的列名完全一致（注意大小写和空格）。

### Q4：图表无法生成

**常见原因：**

1. **变量类型不适合当前图表**：例如对文本变量尝试生成直方图会失败。系统已做类型过滤，如果某变量图表缺失，检查其推断类型是否正确（Tab 3）。
2. **所有值均为缺失**：如果变量全部为 NaN，图表无法生成。检查 Tab 2 的缺失值统计。
3. **类别过多**：分类变量如果唯一值超过 20 个可能被识别为 `high_cardinality`，建议合并类别后重新上传。

### Q5：报告仍然出现旧项目措辞

**验证方法：**

在生成的 HTML 报告中搜索以下关键词，**任何一项都不应出现**：
- `政务`
- `政务服务中心`
- `公众满意度`
- `district`
- `channel`
- `wait_time`
- `SPSS_Data`
- `public_service_variable_diction`

如果出现上述措辞，说明进入了示例模式而不是通用模式。确保在侧边栏顶部选择了 **「通用模式（任意问卷数据）」**。

### Q6：Streamlit 启动报错 / 页面空白

1. **依赖缺失**：运行 `pip install -r requirements.txt` 确保所有依赖已安装。
2. **端口冲突**：`streamlit run app.py --server.port 8502` 指定其他端口。
3. **Python 版本**：需要 Python 3.9+。
4. **查看终端日志**：Streamlit 的错误信息会打印在启动终端中，查看具体报错定位问题。

| **单元测试** | 后端模块、数据处理、统计分析、安全工具 | `python -m pytest tests/ -v`（不含 AppTest） |
| **集成测试** | CSV 加载 → 变量推断 → 分析 → 报告生成 | `python test_run4.py` |
| **AppTest 前端烟雾测试** | Streamlit 页面加载、控件存在、session_state | `python -m pytest tests/test_streamlit_app_smoke.py -v` |
| **手动浏览器验收** | 真实 UI 交互、视觉效果、浏览器兼容性 | 按 [`docs/frontend_acceptance.md`](docs/frontend_acceptance.md) 清单逐项检查 |
| **发布检查** | 文件完整性、版本一致性、安全检查 | `python scripts/release_check.py` |

> **说明：** AppTest 是 Streamlit 内置的模拟测试框架（`streamlit.testing.v1.AppTest`），
> 不是完整浏览器 E2E。它能验证页面加载、控件存在和基本交互，但无法测试 CSS 渲染、
> 图表显示和真实浏览器兼容性。真正发布前仍需按
> [`docs/frontend_acceptance.md`](docs/frontend_acceptance.md) 进行人工浏览器验收。

## 自动化测试

项目附带了自动化测试脚本：

```bash
# 单元测试 + AppTest 前端烟雾测试（v0.1.0）
python -m pytest tests/ -v                        # 全部 pytest 测试（687 passed, 0 failed）
python -m pytest tests/test_streamlit_app_smoke.py -v  # 仅 AppTest 前端烟雾测试（15 passed）
python test_run4.py                               # 集成测试（73/73 通过）
python scripts/release_check.py                   # 发布检查（28 passed, 1 warning, 0 failed）
```

每个脚本最后会输出测试结果摘要。

历史测试脚本（第一至三轮）已归档至 `archive/legacy_tests/`，可单独运行但不作为 CI 的一部分：
```bash
python archive/legacy_tests/test_run.py   # 场景一：中文变量 + 含变量说明表（66 项检查）
python archive/legacy_tests/test_run2.py  # 场景二：英文变量 + 无变量说明表（60 项检查）
python archive/legacy_tests/test_run3.py  # 场景三：英文变量 + 缺失值 + 乱码容错（82 项检查）
```

> **注意**：自动化测试脚本直接调用 `src/` 模块，不依赖 Streamlit 界面。测试通过不代表 UI 完全正常，建议仍按上述清单手动走一遍界面流程。
