# 前端手动验收文档 v0.1.0

本文档指导你通过浏览器手动验收 Streamlit 前端页面的全部核心功能。
自动化测试（pytest / AppTest）覆盖了后端逻辑和页面加载，但最终 UI 交互
仍需人工在真实浏览器中确认。

---

## 一、启动方式

### Windows
```bat
run_app.bat
```

### macOS / Linux / 通用
```bash
streamlit run app.py
```

### 浏览器访问
打开浏览器访问：**http://localhost:8501**

## 四、与自动化测试的关系

| 测试类型 | 覆盖范围 | 运行方式 |
|----------|----------|----------|
| pytest 单元测试 | 后端模块、数据处理、统计分析 | `python -m pytest tests/ -v` |
| test_run4.py | 集成测试（CSV 加载 → 分析 → 报告） | `python test_run4.py` |
| AppTest 烟雾测试 | Streamlit 页面加载、控件存在、session_state | `python -m pytest tests/test_streamlit_app_smoke.py -v` |
| **手动浏览器验收**（本文档） | 真实 UI 交互、视觉效果、浏览器兼容性 | 按本文档清单逐项检查 |
| release_check.py | 文件完整性、版本一致性、安全检查 | `python scripts/release_check.py` |

> **说明：** AppTest 是 Streamlit 内置的模拟测试框架，不是完整浏览器 E2E。
> 它能验证页面加载、控件存在和基本交互，但无法测试 CSS 渲染、图表显示和
> 真实浏览器兼容性。因此真正发布前仍需按本文档进行人工浏览器验收。
