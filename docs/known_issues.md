# 已知问题与限制（Known Issues）

本文档列出当前版本（v0.1.0）的已知限制和注意事项，**不代表 Bug**，而是当前设计范围或技术约束。

---

## 1. 中文学术数据库未直接接入

当前版本**不直接接入** CNKI（知网）、万方、维普等中文学术数据库。原因：
- 这些数据库通常需要机构订阅或 IP 认证才能访问 API
- 多数没有公开的、免费的 REST API

**当前方案：** 文献检索主要依赖 Semantic Scholar、OpenAlex、CrossRef 等国际开放学术数据库，中文文献覆盖有限。

**后续计划：** v0.2.0 计划支持手动导入中文文献（BibTeX / RIS / CSV），见 [路线图](roadmap.md)。

## 2. 文献检索以英文数据库为主

Semantic Scholar、OpenAlex、CrossRef 的文献索引以英文出版物为主，中文期刊和学位论文的覆盖范围和元数据质量参差不齐。

**建议：** 如需高质量中文文献，请配合使用 CNKI/万方/维普的专业检索工具，待未来版本支持手动导入后整合。

## 3. 无内置用户认证

当前版本**没有用户登录/认证机制**。Streamlit 应用启动后任何能访问对应端口的人都能使用全部功能。

**影响：**
- 不适合直接在公网处理真实敏感数据
- 多用户通过同一端口访问时会共享 session_state（存在数据交叉风险）

**建议：**
- 本地使用无需额外配置
- 如需内网共享，建议配合 Streamlit 的密码功能或反向代理加 HTTP Basic Auth
- 公网部署前**必须**自行添加认证层

## 4. 无数据库持久化

当前版本**所有状态存储在 Streamlit session_state 中**（内存）。浏览器刷新、服务器重启、长时间不活动后 session 过期，所有未导出的数据和分析结果会丢失。

**不影响：**
- 已下载到本地的报告文件
- 已保存的 API Key 设置（通过 `config/user_settings.json` 持久化）

**后续计划：** v1.0.0 计划引入可选的数据库后端。

## 5. 多用户隔离尚未实现

多人同时访问同一 Streamlit 实例时，所有用户共享相同的应用状态，可能产生数据交叉或互相覆盖。

**建议：** 当前版本仅适合单用户本地使用，或在受控的小团队环境中串行使用。

## 6. 公网部署需自行加固

如果将本应用部署到公网（如云服务器、Streamlit Community Cloud 公开链接），请注意：
- 需自行添加**认证**（Streamlit secrets 密码或反向代理 Basic Auth）
- 需配置 **HTTPS**（Streamlit Cloud 默认提供，自部署需配置 Nginx + SSL 证书）
- 需设置**访问控制**（IP 白名单、防火墙规则）
- 详见 [安全说明](security.md) 中的部署风险矩阵

## 7. AI 报告仅作辅助

AI 生成的报告和解读**不能替代**：
- 人工统计解释与校验
- 领域专业知识判断
- 政策决策依据

AI 输出可能存在以下问题：
- 幻觉（生成不存在的统计数字或文献）
- 统计术语使用不当
- 忽略重要的分析方法限制

**建议：** 始终人工审阅 AI 报告，将 AI 产出视为"初稿辅助"而非"最终成果"。

## 8. 复杂数据设计支持有限

当前版本的分析方法主要针对**简单随机抽样**的横截面问卷数据。对以下场景支持有限或未覆盖：
- 复杂抽样设计（分层抽样、整群抽样、多阶段抽样）
- 加权调查数据（需要 survey weights 的分析方法）
- 纵向数据 / 追踪调查（重复测量、面板数据分析）
- 时间序列数据

**后续计划：** v1.0.0 计划增加加权数据处理支持。

## 9. Bun not found（Claude Code 环境）⚠️ 非应用 Bug

在 Claude Code 环境中操作 git 时，可能看到 `Bun not found` 警告。这是 Claude Code 的 stop-hook 脚本依赖 Bun 运行时导致的，**与本应用无关**。

- **不影响**：应用运行、测试、报告生成、任何业务功能
- **不影响**：正常的 git 操作（commit、push 等均正常完成）
- **根本原因**：Clawd on Desk 的 git hook 回调脚本使用 Bun，而你环境中未安装 Bun
- **是否阻塞 v0.1.0 发布**：❌ 不阻塞。这是开发环境问题，不是应用问题。

## 10. 推荐使用环境

当前版本**强烈推荐**以下使用方式：
- ✅ 本地单机使用（`localhost:8501`）
- ✅ 受控内网环境（小团队共享）
- ⚠️ Streamlit Community Cloud 私有应用（需自行设置认证）
- ❌ 直接暴露在公网（无认证、无 HTTPS）
